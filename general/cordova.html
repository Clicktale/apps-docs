Cordova Integration

Cordova

Clicktale supports tracking and recording the screen for apps built with Cordova. In order to get started with Cordova, you will need to add the plugin as such:

cordova plugin add https://github.com/Clicktale/cordova-plugin-clicktale.git

Currenly we support Android and iOS only via Cordova and the examples below apply to both platforms.

Getting Started

In order to start the Clicktale data collection and recording, please call

ClicktalePlugin.startClicktale("[ACCESS KEY]", "[SECRET KEY]", success, failure);

In all the code exmpales, success and faliure are function callbacks from the Clicktale - Cordova bridge, giving you some feedback if the API call succeeded or not

    var success = function() {}

    var failure = function(errorCode) {}

Tracking Events

It is possible to track events via that following JS APIs:

ClicktalePlugin.trackEvent("CORDOVA EVENT", success, failure);

ClicktalePlugin.trackEventAndValue("CORDOVA EVENT", "DEVICE READY" ,success, failure);

Tracking Page-Views

Use the following to track the Page View event:

ClicktalePlugin.trackPageView("CORDOVA PAGE", success, failure);

Setting the User ID

To track the session user ID, use the following API:

ClicktalePlugin.setSessionUserID("USER ID", success, failure);

Handling Privacy

In cases where you would like to pause the recording or hide the entire screen, use the following APIs:

ClicktalePlugin.startHidingScreen(success, failure);
// Or to pause at the current frame:
ClicktalePlugin.pauseScreenRecording(success, failure);

To stop hiding the screen or resume recording:

ClicktalePlugin.stopHidingScreen(success, failure);
// Or to resume after recording was paused:
ClicktalePlugin.resumeScreenRecording(success, failure);

Recording lifecycle callbacks

If you would like to hook on the recording start events that fire when the Clicktale recorder begins, you can use the following addEventListener calls:

document.addEventListener( 'onClicktaleSessionURLCreated' ,this.ClicktaleSessionURLCreated);

document.addEventListener( 'onClicktaleSessionIDCreated'  ,this.ClicktaleSessionIDCreated);

Then the implementation of these functions can be something like:

ClicktaleSessionURLCreated: function(e) {
    var success = function() {}

    var failure = function(errorCode) {}

    ClicktalePlugin.trackEventAndValue("CORDOVA EVENT - SessionURL", e.URL ,success, failure);
},

ClicktaleSessionIDCreated: function(e) {
    var success = function() {}

    var failure = function(errorCode) {}

    ClicktalePlugin.trackEventAndValue("CORDOVA EVENT - SessionID", e.SessionID ,success, failure);
}

Getting the session link

In events where the lifecycle callbacks are not sufficient for your implementation, it is also possible to get the session link using a specific API - this link will later send you directly to the video recording of the session that is currently being recorded.

ClicktalePlugin.getSessionLink(success, failure);

Disabling Clicktale's crash reporter

In some cases, your app may already be using a crash reporter and to avoid conflicts, it may be required to turn off Clicktale's crash reporter, you can do that using the following API:

ClicktalePlugin.setCrashReporterOff(success, failure);

Debug Mode

In order to enable debug mode which outputs verbose logging to the console, you can use the following API:

ClicktalePlugin.setDebugLogLevelOn(true, success, failure);

It is also possible to set the videos to be saved into the device media library, using the following API:

ClicktalePlugin.setVideoTestMode(true, success, failure);

Note, do not release an app with that setting enabled, as it will store the recorded videos on your uses devices.
