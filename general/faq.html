FAQ

Data and Performance

How will Clicktale SDK affect my App performance?

It will not. Please look at the Performance article for more information.

When is data sent by the SDK

We record data and process it on the device only after the user has left your app. We will then send this data back to the server.

Will the SDK use my users data plan

The video session data can be configured to be sent over WiFi only or cellular only, or both.

You are in control on how it behaves, and this configuration can be done remotly so no redployment of the app is required if you change your mind.

How much data is sent

The data transmitted depends on the quality of the recording – for the lowest quality the video will be around 100-300KB per minute.

For the higher quality, the video can get up to 1MB per minute.

It is important to understand that the video compression used can vary between different applications, depending on the actual content that displayed on the screen.

Where do I go to add Android or iOS app

When we define an 'app' inside our platform, it is both for Android/iOS. So when you initially log in and look at the dashboard view for your app, it will contain data from both iOS and Android.

Our platform knows how to segment the data correctly, both for heatmaps and any other segmentation that is done on the system.

Usually we create more than one project for your app, to segment data between development and production, but each one of these projects will hold data like any other from your entire app ecosystem.

Privacy

When it comes to privacy, Clicktale offers several options to hide sensitive information from the recording, more details on how to implement the data hiding can be found at the respected plaform documents (Android / iOS)

Is it possible to hide sensitive fields from the recording

Yes, it is possible to completely block any UI element from the recording in both iOS and Android. More information can be found here for Android or iOS

Is data set as sensitive completely blocked or is masked at the Clicktale for Apps application / server

The data set as sensitive is completely blocked. This data is rendered as black areas on the client side and is never sent to Clicktale servers.

What is private by default

By default, all sensitive fields (password fields) are automatically hidden from the recording.

The rest of the data is visible unless manual action is taken to block it – so if your application has sensitive data such as credit card information or any data that you would like to hide from the videos sent to Clicktale Servers – follow the articles around Privacy for any View to ensure the right data is blocked.

Additionally, the keyboard is not recorded, so when a keyboard is displayed on the screen, we will render part of the screen black where the keyboard was present.

Is the data collected by the SDK shared with any third party

No. The data collected by the SDK is sent to Clicktale servers and is never shared with an external source.

Is it possible to pause the recording / black out the entire screen

Yes it is, in the privacy articles (Android or iOS) there are APIs around the different controls you have around recording. It is possible to hide the entire screen with a single api or pause the recording completely.

How do I properly record login screens provided by SDKs such as Google login

Login SDKs usually show a WebView in order to provide functionality, Clicktale for Apps cannot ‘see’ what is happening inside that web view and thus cannot automatically hide the password fields showing in these screens. It is strongly recommended to set these views to private to avoid recording your customers private information.
