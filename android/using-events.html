Using Events

Using Events with Android

Clicktale for Apps SDK enables you to send events to track various business states in your application. Events are later used to segment and analyze your data.

Setting the UserID

The UserID is an event you can use to help you understand and match which user as you know it has experienced what you see in the video recording.

You should use this method when the User identity is updated, either login / logout or at other steps where the unique ID of a user is set.

It is suggested to use UserID in order to allow you to track the same user across platforms or between server and client side events.

Example:

 Clicktale.setUsedId("userID");

Parameters:

UserID: The applications-specific user identifier. Can be any alphanumeric value.

Tracking Events

Events are all the actions that occur in your app, and you want to measure. Each action has one or more related metrics that are incremented every time the event occurs.

For example, you can track events every time an item added to cart; a category selected, or an error message appeared.

Example:

Clicktale.logEvent("event_name", "value");

Parameters:

event_name : The name of the event (e.g. "AddToCart")

value: A value is a specific data point for an event (e.g. "$3.72", "apple")

Tracking Page Views

Every time a new Activity is displayed in your application, it is being tracked automatically by the SDK which will write a page view automatically at that time.

In some cases, for example when a screen logical state changes without loading a new Activity, or because you choose to deactivate the automatic detection completely, you can always track page views manually using the following API call.

Example:

 Clicktale.logPageView("ScreenName");

Parameters:

ScreenName: The name of the screen (e.g. "Product Screen").

Tracking Web Events

We provide an API that enables applications that contain web pages (WebView only) to track events from them.

1.First, on the application side you need to bind your WebView using this API:

Clicktale.bindWebViewForEventTracking(mWebView);

2.Then, on the web page you can use the following JavaScript API's to tag events:

 //Tracking an event without a value:
 Clicktale.TrackEvent('event name here');

 //Tracking an event with a Value
 Clicktale.TrackEventAndValue('event name here','event value here');

 //Tracking a page view
 Clicktale.TrackPageView('page name here');
