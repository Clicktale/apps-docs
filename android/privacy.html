Privacy

Privacy

As our SDK will record everything displayed on your clients screens, it is important to have the ability to hide sensitive information or presonally identifyable information (PII).

The view containing that information will appear as a "black box" in the rendered video:

Hide Native Elements

The SDK automatically hides text fields which are defined as password fields.

In order to hide views that are not defined as password fields and therefore are not hidden automatically, you can send the view to the function hideView() as shows below.

Clicktale.hideView(View view);

For example:

Clicktale.hideView(findViewById(R.id.view_to_hide));

The list of sensitive views is cleaned up automatically on the Activity's onPause method, so calling Clicktale.hideView() should be done inside or after the Activity's onResume method.

Alternatively, in cases where data is loaded dynamically and at one point is no longer sensitive, you can use the following API to un-hide a view

Clicktale.unHideView(View view);

In other cases, where you know that all elements are no longer sensitive in the current Activity or Fragment, you can call the following to immediately clean up all the hidden views:

Clicktale.clearHiddenViews();

Working with a single Activity and multiple Fragments

Clicktale recommends to always clean up the list of hidden views when working with multiple Fragments on the same Activity. While by default, we will do the cleanup automatically in the Activity onPause call - when a single Activity is live during the lifetime of the session, and only multiple Fragments are used, it could be that some recording errors will occur if the list is not maintained by the developer - for single Activity, multiple Fragments cases.

Hide the entire screen

It is possible to hide the entire screen, if activated, the recording will show a black screen until deactivation:

  .......
  //In any point that you do not want to capture the screen
  Clicktale.startHidingScreen();
  .....

Do not forget to stop hidding

  .......
  //In any point that you want to resume capturing the screen
  Clicktale.stopHidingScreen();
  .....

In addtion you can check whether the screnn is hidden 

  .......
  Clicktale.isScreenHidden();
  .....

Hide Web Elements (WebView)

In order to support hiding specific elements inside a WebView, the following is required:

  1. WebView should be JavaScript enabled, in case this is not yet configured as such, it can be done by accessing the WebView settings as such: 
     mWebView = (WebView) findViewById(R.id.webView);
 WebSettings webSettings = mWebView.getSettings();
 webSettings.setJavaScriptEnabled(true);
  2. Hiding elements inside a WebView is only supported from Android SDK level 19.
  3. Requires Clicktale for Apps SDK version 2.0 and up.

In case that the sensitive element is located inside a web page which is displayed by the app:

  1. Declare the containing WebView by calling Clicktale.setWebView(WebView webView).

    The WebView is cleaned automatically from being 'private' on the Activity's onPause method, so calling Clicktale.setWebView() should be done inside or after the Activity's onResume method.

     Clicktale.setWebView(mWebView);

    In order to stop hiding elements inside the WebView altogether, pass null to the setWebView() function:

     Clicktale.setWebView(null);
  2. In the webpage that the WebView will display, give the element that you'd like to hide a "ctHidden" class.
     <input id="pass1" type="text" name="password" class="ctHidden">

Hiding web elements inside Fragments

When using Fragments, it is best practice to set the WebView on the Fragment's onResume and clean it manually on the Fragment's onPause (by passing null to Clicktale.setWebView). This will instruct the SDK to stop bridging into the WebView when not required as the Fragment is no longer visible.

Clicktale.setWebView(null);

Stop and clear all PII

You can clear all the PII list on the SDK with this method:

Clicktale.clearHiddenViews();

Pause Screen Recording

There is an ability to pause and resume acreen recording

Pause Screen Recording:

Clicktale.pauseScreenRecording();

Resume Screen Recording:

Clicktale.resumeScreenRecording();

Check if the screen is Recorded:

Clicktale.isScreenRecorded();
