Push Notifications

Push Notifications

When working with push notifications, many of our customers see value by tracking push notifications opens - and segmenting according to data that was created by that push message - meaning sessions which began after the open of the message.

This helps our customers to analyze the impact of a push message by watching the video recordings of such sessions and looking at heatmaps of data which was generated by the push message.

Tracking Push Notifications

On Android, first we have to identify if the Activity was started from notification: When creating the notification the action it holds is defined by a PendingIntent containing an Intent that may start an Activity.

Let’s see an example of supplying this Intent with an identifier. We will later check this identifier when the Activity starts.

// When creating the notification, creating the Intent for the PendingIntent:
Intent intent = new Intent(context, MainActivity.class);
intent.putExtra("fromIntent", true);

...

// When the Activity starts:
if (getIntent().getBooleanExtra("fromIntent", false)) {
   Clicktale.logEvent("RemoteNotificationReceived");
}

It may be useful to extract send send some data about the message that would make it easy to track (like a push message id, a title or anything that would have some business meaning) as the value (the second argument of Clicktale.logEvent) is search-able inside of the Clicktale for Apps platform.

Clicktale.logEvent("RemoteNotificationReceived", payload.getId());

Also note, that we only track the opening of a push message, and not the arrival of one, this is because we are looking to link a session recording to the data, and there is no session unless the push message was opened.
