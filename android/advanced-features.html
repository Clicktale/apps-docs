Advanced Features

Controlling Crash Reporting

In some cases there is already an existing crash reporting solution and Clicktale's may conflict with it - to enable the ability to have both solutions you can disable Clicktale's crash reporting functions.

When disabled, the recording will still work, and any data integration which enables you to see the Clicktale session link from the existing crash reporting solution will still work as expected.

To disable crash reporting:

Clicktale.setCrashReporterOff();

You must call this function before calling Clicktale.start.

Session Location

Clicktale for Apps enables you to see the location of where the session was recorded at, in case your application uses location services and collects the user location, you can can send the location to the SDK and it will be displayed in the dashboard.

Use the following API to track location:

Clicktale.logLocation(Location location);
