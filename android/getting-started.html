Getting Started

Getting Started with Android

Adding the library to your project

Add .jar file to the libs folder

Go to your project’s root folder (usually “app”) from Finder/File Explorer. Create a folder named “libs” in the root if it doesn’t already exist, and copy the “clicktale.jar” file to this folder.

Update the 'build.gradle' file

Open your Android Studio Project, and go to the “build.gradle” file of your project.

Here, add the following lines to the “dependencies” section:

dependencies { 

    compile files('libs/clicktale.jar') 
    compile 'com.google.code.gson:gson:2.4' 
    … 
    …
}

Update the manifest.xml file

Clicktale Android SDK is using a service to create and upload the session playback.

Declare the CTService in your manifest:

<service android:enabled="true" android:name="com.clicktale.clicktalesdk.CTService"/>

In case there is no Wi-Fi available, a receiver/job scheduler will trigger the video uploading when the Wi-Fi connection will be established (JobScheduler will be used for API LOLLIPOP onwards).

Add a declaration for the CTJobService and the CTWifiReceiver in your manifest:

<service android:name="com.clicktale.clicktalesdk.CTJobService"
    android:permission="android.permission.BIND_JOB_SERVICE"/>

<receiver android:name="com.clicktale.clicktalesdk.CTWifiReceiver" android:enabled="false">
    <intent-filter>
        <action android:name="android.net.conn.CONNECTIVITY_CHANGE"/>
    </intent-filter>
</receiver>

This receiver/job scheduler will register itself only in the case it is needed and unregister afterwards.

Also, make sure that the INTERNET & ACCESS_NETWORK_STATE permission is set:

<uses-permission android:name="android.permission.INTERNET"/>
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE"/>

We use the ACCESS_NETWORK_STATE permission in order to determine whether to upload playback videos over WIFI or Cellular according to the active configuration value.

Authentication and Recording

In order to ensure that only you can send data to your project, you are required to send your access key and secret key to authenticate with our server before you begin recording.

In the Application class, add the following line to your onCreate method, if your app does not have an application class look at the end of this page for instructions on how to set it up.

Clicktale.start(Application, “access_key”, "secret_key");

Unlike in our iOS SDK, the authentication and recording is wrapped into a single function, so calling that function will also start recording.

Another option is to call the start() function with a callback, for example:

public class App extends Application implements ClicktaleCallback { 
    @Override 
    public void onCreate() { 
        super.onCreate(); 
        Clicktale.start(this, this, “access_key”, "secret_key"); 
    }

    @Override 
    public void onSessionURLCreated(String sessionLink) { 
        //Optional implementation 
    } 

    @Override 
    public void onSessionIDCreated(String sessionID) { 
        //Optional implementation 
    }
}

    import android.app.Application; 
public class App extends Application {

  public void onCreate() { 
      super.onCreate(); 
  }
}

  3. Open the “AndroidManifest.xml” file and add a reference to “App” in the android:name attribute of the application tag. The manifest file will look similar to this one:

    <?xml version="1.0" encoding="utf-8"?>
<manifest xmlns:android="http://schemas.android.com/apk/res/android" package="com.example">
<application
    android:name=".App"
    android:allowBackup="true"
    android:icon="@mipmap/ic_launcher"
    android:label="@string/app_name"
    android:theme="@style/AppTheme">
    <activity android:name=".MainActivity">
        <intent-filter>
            <action android:name="android.intent.action.MAIN" />
            <category android:name="android.intent.category.LAUNCHER" />
        </intent-filter>
    </activity>
</application>
</manifest>
