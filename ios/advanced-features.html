Advanced Features

Advanced Features

Controlling Crash Reporting

In some cases there is already an existing crash reporting solution and Clicktale's may conflict with it - to enable the ability to have both solutions you can disable Clicktale's crash reporting functions.

When disabled, the recording will still work, and any data integration which enables you to see the Clicktale session link from the existing crash reporting solution will still work as expected.

To disable crash reporting on iOS:

-(void)setCrashReporterOff;

Or with Swift:

open func setCrashReporterOff()

Call that function before called startClicktale if you do not want the SDK to collect crash information.

Debug Mode

When developing, in order to get immediate feedback on how the SDK captures data, specifically how the video look, you can activate debug mode. We find this being used the most when testing is PII is hidden correctly.

When active, this will cause the videos to be saved to the Photos Album on the device, before they are uploaded to our server.

In order to activate debug mode, use the following function:

-(void)setDebugMode:(BOOL)on;

Or with Swift:

open func setDebugMode(_ on: Bool)

Calling it with true will activate it, false will deactivate it.

This will require access permission to the camera roll - if that permission is not required for your application it is recommended to remove it after the development cycle is complete

Session Location

Clicktale for Apps enables you to see the location of where the session was recorded at, in case your application uses location services and collects the user location, you can can send the location to the SDK and it will be displayed in the dashboard.

Use the following API to track location:

-(void)trackGeoLocation:(CLLocation *)location;

Or with Swift:

open func trackGeoLocation(_ location: CLLocation!)

Improving data delivery

Clicktale recommends sending the video data over WiFi to avoid any data charges for your users. Depending on your app usage, there may be scenarios where it will not be used in WiFi which means the amount of videos you will be able to capture will be much lower than actual recordings.

In order to improve data and video delivery to Clicktale you can use the Background Fetch mechanism, the apple documentation on that API can be found here.

If you are already using background execution, add a call to the following function provided by the Clicktale SDK, this should be placed in the performFetchWithCompletionHandler which should already present in the AppDelegate file

-(void)flushDataInBackgroundWithCompletionHandler:(void (^)(UIBackgroundFetchResult result))completionHandler;

Or with Swift:

open func flushDataInBackground(completionHandler: ((UIBackgroundFetchResult) -> Swift.Void)!)

Paramater: completionHandler

If you use the background fetch for just Clicktale, you can pass the completionHandler - the Background Fetch completionHandler you recived from performFetchWithCompletionHandler, and upon data transfer complete by Clicktale, we will call the complition handler for you.

If you use background fetch for additional updates pass nil completionHandler. In this case, upon data transfer complete we will post a notification: kCTClicktaleBackgroundDataFlushFinish.

At that point you should call the completionHandler yourself, the Clicktale code will not do it.

A full use of the mechanism can be found in our demo app.

Implementing Background Execution

If you are not using Background Execution yet, you will need to add the performFetchWithCompletionHandler function to your AppDelegate:

-(void)application:(UIApplication *)application performFetchWithCompletionHandler:(void (^)(UIBackgroundFetchResult result))completionHandler;

Or with Swift:

optional func application(_ application: UIApplication, performFetchWithCompletionHandler completionHandler: @escaping (UIBackgroundFetchResult) -> Void)

Part of enabling background execution will require you to also enable the background fetching via adding the following to your didFinishLaunchingWithOptions:

[[UIApplication sharedApplication] setMinimumBackgroundFetchInterval:UIApplicationBackgroundFetchIntervalMinimum];

Or with Swift:

 UIApplication.shared.setMinimumBackgroundFetchInterval(UIApplicationBackgroundFetchIntervalMinimum)

This call to setMinimumBackgroundFetchInterval need to be performed from the AppDelegate at:

-(BOOL)application:(UIApplication *)application didFinishLaunchingWithOptions:(NSDictionary *)launchOptions;

Or with Swift:

func application(_ application: UIApplication, didFinishLaunchingWithOptions launchOptions: [UIApplicationLaunchOptionsKey: Any]?) -> Bool

Finally, you need to enable (check the box) Background fetch in the Capabilities tab of your application.

The full description of the implementation can be found at the Apple documentation for background execution.

Using Push Notifications to Improve Data Delivery

In addition to the above article, you can improve data delivery by utilising the Silent Push Notification mechanism.

The full description of the implementation can be found at the Apple documentation for Using Push Notifications to Initiate a Download.

You can call this method:

-(void)flushDataInBackgroundWithCompletionHandler:(void (^)(UIBackgroundFetchResult result))completionHandler;

Or with Swift:

open func flushDataInBackground(completionHandler: ((UIBackgroundFetchResult) -> Swift.Void)!)

From the AppDelegate method as such:

- (void)application:(UIApplication *)application didReceiveRemoteNotification:(NSDictionary *)userInfo fetchCompletionHandler:(void (^)(UIBackgroundFetchResult result))completionHandler
{
 //Support for silent push notification
 [[Clicktale sharedInstance] flushDataInBackgroundWithCompletionHandler: completionHandler];
}

Or with Swift:

 func application(_ application: UIApplication, didReceiveRemoteNotification userInfo: [AnyHashable : Any], fetchCompletionHandler completionHandler: @escaping (UIBackgroundFetchResult) -> Void) {
  //Support for silent push notification
  Clicktale.sharedInstance().flushDataInBackground(completionHandler: completionHandler)
 }

Paramater: completionHandler

If you use the Silent Push Notification for just Clicktale, you can pass the completionHandler - the Fetch completionHandler you recived from didReceiveRemoteNotification, and upon data transfer complete by Clicktale, we will call the complition handler for you.

If you use Silent Push Notification for additional updates pass nil completionHandler. In this case, upon data transfer complete we will post a notification: kCTClicktaleBackgroundDataFlushFinish.

At that point you should call the completionHandler yourself, the Clicktale code will not do it.
