Mixpanel

Mixpanel Integration

When using both Clicktale for Apps and Mixpanel, you can now integrate the system and be able to see for each event sent to Mixpanel, a property leading to the corresponding session inside Clicktale.

In order to activate the integration, you will be using Mixpanel’s ‘Super Properties’ feature as described here: https://mixpanel.com/help/reference/ios/#super-properties

In Clicktale for Apps, when a session begins, which may be on first activation (‘splash screen’ like) or when resuming from background after some time of inactivity, a delegate is used to signal that a new session has begun. At that point you should be sending Mixpanel the super property to assign to all subsequent events automatically.

The delegate function should be used like this for iOS:

-(void)clicktaleDidSessionURLCreated:(NSString *)sessionLink
{ 
    Mixpanel *mixpanel = [Mixpanel sharedInstance]; 
    // Send a "Clicktale Session: [url]" property will be sent 
    // with all future track calls. 
    [mixpanel registerSuperProperties:@{@"Clicktale Session": sessionLink}];
}

Or like this for Android:

@Override
public void onSessionURLCreated(String sessionLink) { 
    // Send a "Clicktale Session: [url]" property that will be sent 
    // with all future track calls. 
    MixpanelAPI mixpanel = MixpanelAPI.getInstance(context, MIXPANEL_TOKEN); 
    JSONObject props = new JSONObject(); props.put("Clicktale Session", sessionLink); 
    mixpanel.registerSuperProperties(props);
}

Only use registerSuperProperties. Do not call Mixpanel registerSuperPropertiesOnce, as the session id can change when your app is coming back from background
